# my-awesome-turbowarp-plugins

TurboWarp 自定义扩展插件合集。

当前仓库主要提供一个扩展：

- **百炼大模型（DashScope AI）**：集成阿里云百炼，支持文字对话、摄像头画面保存、图片理解、语音识别（STT）和语音合成（TTS）。

## 文件说明

| 文件 | 说明 |
|------|------|
| `dashscope_ai.js` | TurboWarp 百炼大模型扩展源码 |

## 功能总览

### 1. 文字对话

- 向大模型发送文本问题
- 通过事件积木接收回答
- 通过报告积木读取最近一次回答

### 2. 摄像头画面保存

- 把当前摄像头画面保存为图片文件
- 保存时会同时缓存这一帧，供后续图片理解使用

### 3. 图片理解

- 直接发送当前摄像头画面给视觉模型
- 或发送最近一次保存/缓存的摄像头图片
- 通过事件积木接收图片理解结果

### 4. 语音识别（STT）

- 开始录音
- 停止录音后发送到百炼识别
- 通过事件积木接收识别文本

### 5. 语音合成（TTS）

- 把文字合成为语音
- 播放最近一次合成结果

## 默认模型

| 能力 | 默认模型 |
|------|------|
| 文字对话 | `qwen-flash` |
| 图片理解 | `qwen3.6-flash` |
| 语音识别 | `qwen3-asr-flash` |
| 语音合成 | `cosyvoice-v3.5-flash` |

## 使用前准备

### 1. 获取 API Key

1. 打开[阿里云百炼平台](https://bailian.console.aliyun.com/)
2. 创建或获取 API Key
3. 在 TurboWarp 积木中直接传入 API Key

### 2. 以非沙箱模式加载扩展

1. 将 `dashscope_ai.js` 托管到公开可访问的 URL
2. 打开 [TurboWarp](https://turbowarp.org/)
3. 点击左下角 **添加扩展**
4. 选择 **自定义扩展**
5. 输入 `dashscope_ai.js` 的 URL 并加载

也可以直接将文件内容复制到 TurboWarp 的自定义扩展加载器中。

### 3. 开启摄像头

涉及摄像头保存或图片理解时，需要先让 TurboWarp 项目启用视频设备。

建议先使用 Scratch / TurboWarp 内置视频相关积木开启摄像头，再调用本扩展中的摄像头功能。

### 4. 允许麦克风权限

涉及语音识别时，浏览器会请求麦克风权限，需要允许访问。

## 积木清单

### 文字对话

| 积木 | 类型 | 说明 |
|------|------|------|
| 使用 API Key [KEY] 向模型 [MODEL] 提问 [QUESTION] | 指令 | 发送文字问题 |
| 当收到大模型回复时 | 事件 | 收到文字回复时触发 |
| 大模型的回答 | 报告 | 返回最近一次文字回复 |

### 摄像头 / 图片理解

| 积木 | 类型 | 说明 |
|------|------|------|
| 保存摄像头画面为 [FILENAME] | 指令 | 保存当前摄像头画面，并缓存这一帧 |
| 使用 API Key [KEY] 向视觉模型 [MODEL] 发送摄像头画面并提问 [PROMPT] | 指令 | 直接抓取当前摄像头画面并发送 |
| 使用 API Key [KEY] 向视觉模型 [MODEL] 发送最近一次保存的图片并提问 [PROMPT] | 指令 | 发送最近一次保存或缓存的图片 |
| 当图片理解完成时 | 事件 | 收到图片理解结果时触发 |
| 图片理解结果 | 报告 | 返回最近一次图片理解结果 |

### 语音识别

| 积木 | 类型 | 说明 |
|------|------|------|
| 开始录音 | 指令 | 请求麦克风权限并开始录音 |
| 停止录音并使用 API Key [KEY] 识别语音 | 指令 | 停止录音并调用 STT |
| 当语音识别完成时 | 事件 | 收到识别结果时触发 |
| 语音识别结果 | 报告 | 返回最近一次识别文本 |

### 语音合成

| 积木 | 类型 | 说明 |
|------|------|------|
| 使用 API Key [KEY] 将 [TEXT] 合成为语音 | 指令 | 调用 TTS 合成语音 |
| 当语音合成完成时 | 事件 | 语音合成完成时触发 |
| 播放合成语音 | 指令 | 播放最近一次合成结果 |
| 停止播放语音 | 指令 | 停止当前播放 |

## 使用流程

下面每个流程都可以单独使用。

## 流程一：文字对话

### 适用场景

- 普通问答
- 知识查询
- 文本生成

### 使用步骤

1. 加载 `dashscope_ai.js`
2. 准备好 API Key
3. 放置事件积木 **当收到大模型回复时**
4. 在事件下使用 **大模型的回答**
5. 调用 **使用 API Key [KEY] 向模型 [MODEL] 提问 [QUESTION]**

### 示例

```text
当 🟢 收到大模型回复时
  说（大模型的回答）

使用 API Key [sk-xxx] 向模型 [qwen-flash] 提问 [你是谁？]
```

## 流程二：保存摄像头画面为图片

### 适用场景

- 保存当前摄像头截图
- 为后续图片理解缓存图片

### 使用步骤

1. 在 TurboWarp 中先开启摄像头
2. 确认浏览器已经允许摄像头权限
3. 调用 **保存摄像头画面为 [FILENAME]**
4. 浏览器会下载图片文件
5. 扩展会同时缓存这一帧图片

### 示例

```text
保存摄像头画面为 [capture.png]
```

## 流程三：直接发送当前摄像头画面做图片理解

### 适用场景

- 实时分析当前镜头画面
- 不需要先下载图片

### 使用步骤

1. 在 TurboWarp 中先开启摄像头
2. 放置事件积木 **当图片理解完成时**
3. 在事件中读取 **图片理解结果**
4. 调用 **使用 API Key [KEY] 向视觉模型 [MODEL] 发送摄像头画面并提问 [PROMPT]**
5. 等待视觉模型返回结果

### 示例

```text
当 🟢 图片理解完成时
  说（图片理解结果）

使用 API Key [sk-xxx] 向视觉模型 [qwen3.6-flash] 发送摄像头画面并提问 [请描述当前摄像头画面]
```

## 流程四：先保存图片，再发送同一张图做图片理解

### 适用场景

- 先保存证据图，再分析
- 希望“保存的图”和“发送给模型的图”是同一帧

### 使用步骤

1. 在 TurboWarp 中先开启摄像头
2. 调用 **保存摄像头画面为 [FILENAME]**
3. 该积木会下载图片，同时缓存这一帧
4. 调用 **使用 API Key [KEY] 向视觉模型 [MODEL] 发送最近一次保存的图片并提问 [PROMPT]**
5. 通过 **当图片理解完成时** 读取结果

### 示例

```text
当 🟢 图片理解完成时
  说（图片理解结果）

保存摄像头画面为 [capture.png]
使用 API Key [sk-xxx] 向视觉模型 [qwen3.6-flash] 发送最近一次保存的图片并提问 [请分析刚才保存的图片]
```

### 说明

浏览器下载到本地的图片文件不会再被扩展直接读取回来。

这里“发送最近一次保存的图片”实际发送的是**保存动作发生时缓存下来的同一帧图片数据**，不是重新读取磁盘文件。

## 流程五：语音识别

### 适用场景

- 把语音转成文字
- 后续再接文字问答或语音播报

### 使用步骤

1. 放置事件积木 **当语音识别完成时**
2. 在事件中读取 **语音识别结果**
3. 调用 **开始录音**
4. 等待用户说话
5. 调用 **停止录音并使用 API Key [KEY] 识别语音**

### 示例

```text
当 🟢 语音识别完成时
  说（语音识别结果）

开始录音
停止录音并使用 API Key [sk-xxx] 识别语音
```

## 流程六：语音问答（语音识别 → 文字对话）

### 适用场景

- 语音提问，大模型文字回答

### 使用步骤

1. 放置 **当语音识别完成时**
2. 在事件中调用文字对话积木
3. 放置 **当收到大模型回复时**
4. 在事件中读取 **大模型的回答**
5. 先录音，再执行识别

### 示例

```text
当 🟢 语音识别完成时
  使用 API Key [sk-xxx] 向模型 [qwen-flash] 提问（语音识别结果）

当 🟢 收到大模型回复时
  说（大模型的回答）

开始录音
停止录音并使用 API Key [sk-xxx] 识别语音
```

## 流程七：语音问答（语音识别 → 文字对话 → 语音合成）

### 适用场景

- 语音助手
- 语音提问，语音回答

### 使用步骤

1. 放置 **当语音识别完成时**
2. 在事件中把识别结果发给大模型
3. 放置 **当收到大模型回复时**
4. 在事件中把回答交给语音合成
5. 放置 **当语音合成完成时**
6. 在事件中播放合成语音

### 示例

```text
当 🟢 语音识别完成时
  使用 API Key [sk-xxx] 向模型 [qwen-flash] 提问（语音识别结果）

当 🟢 收到大模型回复时
  使用 API Key [sk-xxx] 将（大模型的回答）合成为语音

当 🟢 语音合成完成时
  播放合成语音

开始录音
停止录音并使用 API Key [sk-xxx] 识别语音
```

## 流程八：文字转语音播放

### 适用场景

- 朗读固定文本
- 播放大模型回答

### 使用步骤

1. 放置 **当语音合成完成时**
2. 在事件中调用 **播放合成语音**
3. 调用 **使用 API Key [KEY] 将 [TEXT] 合成为语音**

### 示例

```text
当 🟢 语音合成完成时
  播放合成语音

使用 API Key [sk-xxx] 将 [你好，世界] 合成为语音
```

## 注意事项

### 1. 运行模式

- 扩展必须在**非沙箱模式**下运行

### 2. 摄像头功能

- 使用摄像头相关积木前，请先开启视频设备
- 如果摄像头未开启，保存和图片理解都会失败

### 3. 语音识别限制

- 默认使用 `qwen3-asr-flash`
- 适合短音频
- 建议将录音控制在约 5 分钟内

### 4. 图片理解默认模型

- 默认使用 `qwen3.6-flash`
- 也可以在积木里手动改成其他兼容视觉模型

### 5. 图片保存与发送的关系

- 保存图片到本地，不等于扩展可以重新读取本地文件
- “发送最近一次保存的图片”依赖扩展缓存的最近一帧数据
